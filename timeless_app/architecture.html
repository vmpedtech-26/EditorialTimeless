Seguridad
Flujo de Token y Validación Offline
Handshake de 30 días que garantiza acceso sin conexión sin comprometer el DRM.
Login y generación de token
Requiere conexión · Una vez cada 30 días
1
Autenticación Firebase
Usuario ingresa credenciales. FirebaseAuth valida y emite idToken JWT.
2
Verificación de suscripción
Cloud Function consulta Firestore: estado de suscripción activa, fecha de expiración.
3
Emisión del DRM Token
Se genera JWT firmado con RS256: contiene uid, device_id, contenido autorizado y exp: 30 días.
4
Derivación de clave local
App deriva clave AES-256 usando PBKDF2(token + device_salt). Clave almacenada en Keystore/Secure Enclave.
5
Contenido descargado y cifrado
Archivos TLF descifrados en servidor → re-cifrados con clave local → SQLite local.
Lectura offline
Sin conexión · Hasta 30 días
1
Verificación local del token
App verifica firma RS256 del JWT localmente. No requiere red. Comprueba exp timestamp.
2
Recuperación de clave
Clave AES-256 recuperada desde Android Keystore o iOS Secure Enclave. Requiere biometría.
3
Descifrado en memoria
Capítulos descifrados chapter-by-chapter en RAM. Jamás se escriben en disco sin cifrar.
4
Guardado de progreso local
Posición de lectura, marcadores y notas en SQLite cifrado. Pendientes de sync.
5
Handshake al reconectar
Al recuperar conexión: refresh token si quedan <7 días, sync de progreso, descarga nuevas obras.

Criptografía
Pipeline de Encriptación
Del archivo TLF en servidor a los bytes en la pantalla del usuario.
1
Servidor: Master Encryption
El TLF del Agente se cifra en Cloud Functions con clave maestra AES-256 gestionada por Google Cloud KMS. Nunca sale de la infraestructura de Firebase.
2
Transmisión: HTTPS + Signed URL
El cliente descarga el TLF cifrado via HTTPS con URL firmada de corta duración (15 min). Imposible redirigir o capturar la URL.
3
Cliente: Re-cifrado con clave de dispositivo
App descifra con clave maestra (token DRM incluido), luego re-cifra inmediatamente con PBKDF2(DRM_token + device_salt). Resultado: AES-256-GCM en SQLite local.
4
Lectura: Descifrado en memoria efímera
Al abrir un capítulo, se descifra en RAM. Nunca se escribe al disco sin cifrar. Al cerrar el capítulo, la memoria se borra explícitamente (zeroing).
5
Revocación: Token inválido → contenido inaccesible
Al revocar la suscripción, el token DRM expira. Sin token válido, la clave PBKDF2 no puede derivarse. El contenido en disco se vuelve ilegible automáticamente.
encryption_service.dart 
// ── Derivar clave desde token DRM ──────
Future<Uint8List> deriveKey(String token) async {
  final salt = await DeviceInfo.getSalt();
  // PBKDF2 · SHA-256 · 100k iteraciones
  return PBKDF2.derive(
    password: token.codeUnits,
    salt:     salt,
    c:        100000,
    dkLen:    32,
  );
}

// ── Cifrar capítulo para caché local ───
Future<EncChapter> encryptChapter(
  Uint8List plaintext,
  Uint8List key,
) async {
  final iv = _randomBytes(12); // 96-bit GCM
  final cipher = AES256GCM(key, iv);
  return EncChapter(
    iv:          iv,
    ciphertext:  cipher.encrypt(plaintext),
    tag:         cipher.tag,
  );
}

// ── Descifrado efímero en lectura ──────
Uint8List readChapter(EncChapter enc, Uint8List key) {
  final plain = AES256GCM(key, enc.iv).decrypt(enc);
  addCleanupCallback(() => plain.fillRange(0, plain.length, 0));
  return plain; // solo en RAM
}
